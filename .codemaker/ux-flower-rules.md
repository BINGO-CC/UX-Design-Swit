
# UX 行为路径输出规则（/FLOWER 模式）

## 执行检查单
每次输出行为路径 JSON 前，必须按顺序完成以下 5 步：

- [ ] Step 1：穷举策划案所有功能模块（逐章节扫描）
- [ ] Step 2：对每个模块判断是否包含用户主动操作行为
- [ ] Step 3：对有操作行为的模块，判断是否已有对应路径，输出覆盖差异表
- [ ] Step 4：实际路径数量 = Step 3 中判定需生成的数量，否则不输出
- [ ] Step 5：路径排列顺序符合下方顺序框架后，再输出完整 JSON

---

## 路径顺序框架

补充路径时，不追加到末尾，插入到对应组的正确位置，输出时始终按此顺序排列：

```
第一组：系统激活与初始化
  └─ 首次激活辉光系统

第二组：魅力值主界面（核心循环）
  ├─ 外观购买 -> 魅力值增长 -> 等级提升
  ├─ 月增长奖励领取
  ├─ 竖向升级路线查看
  └─ 魅力值专属商城购买

第三组：炫耀系统 - 俱乐部
  ├─ 俱乐部大厅日常浏览
  ├─ 图鉴长廊首次进入（形象设置引导）
  ├─ 长廊内修改形象 banner（非首次）
  ├─ 点击用户卡片 -> 名片 -> 社交橱窗
  └─ 当月风尚完整浏览（P1，待补充）

第四组：炫耀公域 - 活动岛（P2）
  ├─ 进入活动岛分层区域
  ├─ NPC 交互路径
  └─ 杂志浏览与互动
```

---

## 适用范围

本规则适用于所有触发 `/FLOWER` 或 `行为路径` 模式的输出任务。
